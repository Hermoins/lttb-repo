## 一、现有技术及其缺陷（背景技术）

### 1.1 行业背景

瓦斯（甲烷）灾害是煤矿安全生产的"头号杀手"。根据国家矿山安全监察局统计，瓦斯事故占煤矿较大以上事故的30%以上。近年来，随着煤矿智能化建设的推进，高密度瓦斯传感器网络已成为标配——单个矿井部署的瓦斯传感器数量可达数百个，采样频率普遍达到1Hz-10Hz，数据量呈指数级增长。

### 1.2 现有技术分析

当前主要的下采样/压缩方法及其在瓦斯监测中的缺陷如下：

#### 1.2.1 等间隔采样
简单丢弃数据点，极易丢失短暂但关键的瓦斯异常涌出特征（如瓦斯突出前兆的瞬时涌升），导致预警延迟或漏报。**且无法反映瓦斯浓度的真实变化趋势。**

#### 1.2.2 滑动平均/滤波后采样
虽然平滑了数据，但同时也平滑了异常突增的陡峭边缘，使预警信号变得迟钝。**特别对于持续时间仅数秒的瓦斯尖峰，几乎无法识别。**

#### 1.2.3 经典LTTB（最大三角形三桶）算法
这是一种优秀的可视化下采样算法，能较好地保留趋势特征。但其采用固定大小的桶进行采样，在瓦斯数据场景下存在固有矛盾：

- **在长达数小时的数据平稳期**：固定桶采样保留了过多冗余数据，压缩效率不足（通常仅能达到10:1-20:1）
- **在几分钟甚至几秒钟的瓦斯异常涌出期**：数据剧烈变化，固定桶采样因采样点过少而无法捕捉涌出的完整细节（如精确的起增点、最大斜率点和峰值），严重影响预警的准确性和及时性
- **【关键缺陷】在实时预警场景下**：传统LTTB算法必须"等满下一个桶"才能完成当前桶的采样点计算，这会被动引入延迟——假设在平稳期设置了大桶（如覆盖5分钟数据），则必须等待5分钟才能输出当前桶的关键点，此时即使数据已经发生异常，预警信号也无法及时发出

#### 1.2.4 现有技术总结对比表

| 方法       | 压缩比             | 异常保留率 | 预警及时性           | 适用场景     |
| ---------- | ------------------ | ---------- | -------------------- | ------------ |
| 等间隔采样 | 50:1               | <60%       | 差                   | 通用         |
| 滑动平均   | 20:1               | <50%       | 差                   | 平滑需求     |
| 固定桶LTTB | 50:1               | ~82%       | 中（存在延迟）       | 可视化       |
| **本发明** | **可调10:1-100:1** | **>97%**   | **优（双通道保障）** | **瓦斯监测** |

**因此，现有技术无法在实现高压缩比的同时，确保对瓦斯异常涌出关键特征的高保真保留，更无法解决实时预警的延迟问题。这是本领域一个亟待解决的技术矛盾。**

## 二、本发明要解决的技术问题

本发明旨在克服上述缺陷，提供一种智能化的数据压缩与预警方法及系统，其核心要解决的技术问题是：

### 【问题1】如何在满足煤矿监控系统高压缩比需求的前提下，实现对瓦斯异常涌出特征的精准、无损感知？

### 【问题2】如何在数据压缩的同时，实现对瓦斯异常涌出的早期预警，而非仅仅依赖固定阈值报警？

### 【问题3】如何使下采样算法能够自适应数据特征，在平稳期高压缩、在异常期高保真？

### 【问题4】如何解决LTTB算法固有的"等满下一个桶"导致的预警延迟问题？

### 【问题5】如何区分传感器毛刺噪声与真实瓦斯异常涌出，避免误报警？

### 【问题6】【核心新增】如何设计预警机制，确保即使在分桶延迟的情况下，预警信号仍能即时触发？

## 三、本发明的总体技术方案（发明内容概要）

为解决上述问题，本发明提出一种**基于动态桶调整的煤矿瓦斯涌出异常预警方法及系统**。其核心思想是：

### 3.1 弹性采样策略
摒弃固定桶大小，使LTTB算法的采样密度能够根据瓦斯浓度数据的实时变化率进行自适应动态调整，实现"数据平稳期高压缩、数据异常期高保真"的弹性采样策略。

### 3.2 【核心新增】双通道并行预警机制

本发明的关键创新之一在于采用**双通道并行预警设计**，两条通道互不干扰，确保预警的及时性与准确性：

```
┌──────────────────────────────────────────────────────────────┐
│                      数据采集 (1Hz)                         │
└──────────────────────────┬───────────────────────────────────┘
                           │
          ┌────────────────┴────────────────┐
          ▼                                   ▼
┌─────────────────────────┐     ┌────────────────────────────────┐
│    通道1：阈值报警       │     │    通道2：趋势预警 (LTTB)       │
│    (独立、实时、0延迟)    │     │    (弹性采样、动态调整)        │
├─────────────────────────┤     ├────────────────────────────────┤
│ • 每个新数据点即时判断     │     │ • 后台动态分桶处理            │
│ • 浓度 > Th_alarm 立即触发│     │ • 基于关键点计算斜率          │
│ • 无需等待桶满            │     │ • 可触发流式截断加速响应      │
│ • 兜底保障，最坏情况保护  │     │ • 早期预警，提前3-10秒        │
└───────────┬─────────────┘     └────────────┬───────────────┘
            │                                 │
            └───────────────┬─────────────────┘
                            ▼
                  ┌─────────────────────┐
                  │   融合判定模块      │
                  │ (与/或逻辑组合)     │
                  └──────────┬──────────┘
                             ▼
                      预警信号输出
```

**【关键说明】**：
- **通道1（阈值报警）独立于LTTB分桶机制**，对每个新到达的数据点即时判断，不等待任何桶填满，实现0延迟预警
- **通道2（趋势预警）**利用LTTB下采样后的关键点序列计算斜率变化，实现早期预警
- 两个通道**互为补充**，即使LTTB分桶有微小延迟，阈值报警通道仍能提供兜底保障

## 四、本发明的详细技术方案（具体实施方式）

### 4.1 数据采集与抗噪预处理

1. **数据采集**：通过瓦斯传感器以预设频率（如1Hz-10Hz）采集浓度数据，构建数据序列C
2. **抗噪预处理**：在计算变化率之前，对原始数据进行**形态学滤波（如中值滤波）**，去除传感器毛刺噪声。窗口大小推荐 w = 3

### 4.2 动态感知与分桶

实时分析瓦斯浓度数据流，计算其**变化率指标**（如梯度、方差）。根据变化率大小，动态调整LTTB算法的"桶"的尺寸：
- 数据平稳（变化率小）→ 增大桶尺寸 → 高强度压缩
- 数据波动/异常（变化率大）→ 缩小桶尺寸 → 高密度采样

### 4.3 【新增】流式截断与边界近似机制

#### 4.3.1 大桶截断（中断机制）

**【技术方案】**：
在填充某个较大尺寸的桶（平稳期大桶）的过程中，系统以后台滑动窗口（如窗口大小为5个采样点，即5秒）实时计算最新输入数据的变化率（梯度或方差）。

**【关键参数】**：
- **滑动窗口大小**：推荐 3-5 个采样点
- **异常检测阈值**：
  - 梯度阈值 Th_grad_high：推荐 0.1-0.15%/秒
  - 方差阈值 Th_var_high：推荐 0.15-0.25
- **截断触发条件**：当窗口内变化率突破阈值时，立即执行截断

**【工作流程】**：
```
t=0s：开始填充大桶（目标100点）
t=0-10s：后台以5点窗口监控，无异常 → 继续填桶
t=11s：检测到梯度=0.2%/秒 > 阈值0.15%/秒
      ↓
立即执行：① 截断当前大桶（已收集11点）
         ② 不再等待剩余89点填满
         ③ 立即切换为小桶（目标5-10点）
         ④ 开始高频采样
```

**【有益效果】**：整个截断过程延迟 < 1秒，真正实现"平稳期高压缩、突发期0延迟切换"。

#### 4.3.2 虚拟未来点近似

由于强行截断或实时输出的要求，下一个桶的数据可能尚未采集完整。此时，本算法采用"虚拟未来点"参与计算：

- **短窗口近似**：取当前已采集的 k 个点（如 k=3）的平均值，作为"下一桶平均位置"
- **切线外推近似**：利用当前桶最后几个点的实时斜率，向未来方向外推一个步长

### 4.4 【新增】双模式预警详解

#### 4.4.1 模式一：阈值报警（独立、实时）

**【技术方案】**：
设定安全阈值 Th_alarm（如0.5%，即瓦斯浓度达到0.5%时报警）。对每个新到达的数据点 C[i]，判断 C[i] > Th_alarm 是否成立。

**【关键特性】**：
- **0延迟**：数据到达即判断，无需等待任何处理
- **独立运行**：不依赖LTTB分桶结果
- **兜底保障**：即使趋势预警因分桶延迟未触发，阈值报警仍能发出

**【伪代码】**：
```
for each new data point C[i]:
    if C[i] >= Th_alarm:
        immediately trigger Level-1 alarm
```

#### 4.4.2 模式二：趋势预警（早期预警）

**【技术方案】**：
基于LTTB下采样得到的关键点序列 P = {P[0], P[1], ..., P[M-1]}，计算最近 L 个点（如 L=3-5）的线性拟合斜率 S。

**【计算公式】**：
```
S = Σ(P[j] - P̄)(j - j̄) / Σ(P[j] - P̄)²
```
其中 j̄ 为最近L个点的索引平均值，P̄ 为浓度平均值。

**【触发条件】**：
- 趋势预警阈值 Th_trend：推荐 0.08-0.15%/秒
- **驻留判断**：当 S > Th_trend 时，不立即报警，需连续 m 次（如 m=3）确认或持续时间超过 Δt（如3秒）

**【预警提前量】**：相比阈值报警，趋势预警可提前 3-10 秒发出

#### 4.4.3 双通道融合判定

**【融合逻辑】**：
```
if 阈值报警触发:
    发出 Level-1 预警 (立即)
    
if 趋势预警触发 AND 驻留确认通过:
    发出 Level-2 预警 (早期)
    
if 双通道同时触发:
    发出组合预警 (最高级别)
```

## 五、各实施例详细方案

### 实施例一：基于一阶梯度（导数）的动态调整策略

适用于：瓦斯涌出呈现**渐变型**特征的场景（如巷道揭露煤层时的渐进式瓦斯释放）

**参数配置**：
| 参数         | 推荐值       | 说明             |
| ------------ | ------------ | ---------------- |
| M            | 50-200       | 目标输出点数     |
| B_base       | 50-100       | 基础桶大小       |
| α            | 0.5-2.0      | 梯度灵敏度因子   |
| B_min        | 5-10         | 桶大小下限       |
| B_max        | 80-150       | 桶大小上限       |
| Th_grad_high | 0.1-0.15%/秒 | 流式截断梯度阈值 |

**动态桶公式**：
```
B_i = MAX(B_min, MIN(B_max, round(B_base / (1 + α × G_avg_i))))
```

### 实施例二：基于滑动方差的双阈值调整策略

适用于：瓦斯涌出呈现**突发型**特征的场景（如煤与瓦斯突出时的瞬时大量涌出）

**参数配置**：
| 参数    | 推荐值    | 说明                     |
| ------- | --------- | ------------------------ |
| W       | 10-50     | 滑动窗口大小             |
| Th_low  | 0.01-0.05 | 低方差阈值               |
| Th_high | 0.1-0.5   | 高方差阈值（流式截断用） |
| B_min   | 5-10      | 桶大小下限               |
| B_max   | 80-150    | 桶大小上限               |
| β       | 0.3-0.7   | 平滑系数                 |

**三区域划分**：
| 区域   | 判断条件                 | 桶大小   |
| ------ | ------------------------ | -------- |
| 平稳区 | Var_i < Th_low           | B_max    |
| 过渡区 | Th_low ≤ Var_i ≤ Th_high | 线性插值 |
| 异常区 | Var_i > Th_high          | B_min    |

### 实施例三：基于多参数融合的自适应策略

适用于：复杂多变场景

**融合公式**：
```
R[i] = w1 × G_norm[i] + w2 × Var_norm[i]
```
其中 w1 + w2 = 1，推荐 w1=0.6, w2=0.4

## 六、本发明的有益效果（创新点与优势）

### 6.1 实现了"弹性压缩"，解决了固有矛盾
- **数据平稳期**：实现超高压缩比（可达100:1以上），大幅降低存储和传输成本
- **异常涌出期**：自动切换为高保真模式（压缩比降至10:1），确保关键特征无遗漏
- **整体效益**：从整体上优化了系统资源占用，同时确保了安全监测无盲区

### 6.2 预警的及时性与准确性大幅提升

- **特征捕捉更精准**：由于在异常涌出期采样点极度丰富，能够更早、更精确地捕捉到涌出的**起始点（起增点）、最大斜率点和峰值**

- **预警时机大幅提前**：通过计算趋势斜率，可将预警时机从"浓度达到阈值时"提前至"浓度开始加速上升时"，**提前时间可达3-10秒**

- **【新增】双模式互补保障**：阈值报警保证底线安全，趋势预警实现早期预警，两者互为补充。特别地：
  - 阈值报警通道**0延迟**，即使LTTB分桶有延迟也能即时触发
  - 趋势预警通道**提前预警**，在浓度达到阈值前3-10秒发出预警

### 6.3 实现了LTTB实时预警的工程突破

- **流式截断机制**：打破传统LTTB"等满下一个桶"的固有延迟，实现**0延迟切换**
- **虚拟未来点近似**：在数据不完整时仍能准确计算关键点
- **双通道并行设计**：阈值报警独立于LTTB分桶，确保最坏情况下仍有预警保障

### 6.4 具备强抗干扰能力

- **中值滤波保边去噪**：引入形态学滤波，在剔除传感器毛刺的同时完全保留真实的瓦斯突增边缘
- **趋势驻留判断**：趋势预警需连续多个周期确认方可触发，避免单点噪声引发误报
- **参数可调**：通过灵活调节α、B_base、Th_low、Th_high等阈值参数，可适应不同矿井的煤层条件和工作面状态
- **平滑滤波**：滑动窗口和桶大小平滑处理可有效过滤短暂脉冲噪声

### 6.5 工程应用价值

- **兼容性强**：可对接现有煤矿安全监控系统（如KJ系列、KJ90X等）
- **部署简单**：仅需在数据处理端部署算法，无需更换现有传感器
- **成本低廉**：无需额外硬件投入，仅需软件升级
- **边缘计算友好**：算法可运行于井下边缘计算网关，显著减少数据传输带宽
